  • 1. Introduction to Super Resolution
  • What is Super Resolution?
  • Real-World Applications
  • What is Super Resolution?
  • Super Resolution (SR) uses Deep Learning to reconstruct a high-resolution image from one or more low-resolution inputs.
  • It aims to recover:
  • • Lost image details & fine textures
  • • Sharp edges and structural information
  • • Overall visual quality
  • Basic Workflow:
  • LR Image → Feature Extraction → SR Model → Reconstruction → HR Image
  • Real-World Applications:
  • • Satellite Imaging — terrain analysis, environmental monitoring
  • • Surveillance — face recognition, number plate enhancement
  • • Medical Imaging — MRI/CT scan enhancement
  • • Mobile Cameras — AI zoom, night photography
  • • Video Upscaling — HD to 4K conversion
  • • Remote Sensing — land use, geological mapping
  • Industry Usage:
  • Samsung, Google, Apple use SR in camera systems, satellite processing, and video restoration.
  • 2. Problem Statement
  • Most open-source data have coarse spatial resolution, and it is advantageous to use different sensors for effective enhancement; however, the sensors have differences in spatial resolution, spectral characteristics, noise levels, and often lack spatial or temporal overlap.
  • Key Challenges in SISR:
  • • Recovering lost high-frequency details from degraded inputs
  • • Balancing reconstruction quality vs. computational cost
  • • Maintaining global context while preserving local pixel fidelity
  • • Existing CNN models have limited receptive fields
  • • Transformer models suffer from quadratic computational complexity
  • • Pure Mamba models lose 2D spatial context due to 1D sequential scanning
  • 3. Traditional Super Resolution Methods
  • Before deep learning, SR relied on interpolation-based image scaling methods:
  • Nearest Neighbour
  • • Copies nearest pixel value
  • • Fast but blocky appearance
  • • Jagged edges, poor quality
  • Bilinear Interpolation
  • • Uses neighbouring pixels
  • • Smoother than nearest
  • • Loss of sharpness, blurred edges
  • Bicubic Interpolation
  • • Uses 16 surrounding pixels
  • • Better edge continuity
  • • Still cannot reconstruct missing details
  • Limitation: All interpolation methods fail to reconstruct truly missing high-frequency details → Need for Deep Learning
  • 4. Deep Learning in Super Resolution
  • Why Deep Learning?
  • Evolution of SR Architectures
  • Traditional methods cannot recover missing information.
  • Deep Learning models learn the LR→HR mapping directly from large datasets.
  • What DL Models Learn:
  • • Texture patterns (roads, vegetation, faces)
  • • Edge information (boundaries, transitions)
  • • Spatial structures (relationships between regions)
  • • Hierarchical features (low-level → high-level)
  • 1. CNN-based → SRCNN, FSRCNN, VDSR
  • Local feature extraction, fast but limited receptive field
  • 2. GAN-based → SRGAN, ESRGAN
  • Perceptual realism, but training instability
  • 3. Transformer-based → SwinIR
  • Global attention, but O(n²) complexity
  • 4. State Space Models → SR-Mamba
  • Linear complexity, but 1D scan loses 2D context
  • 5. Hybrid → SRMamba-T (Ours)
  • Best of Mamba + Transformer!
  • 5. CNN-based Models for Super Resolution
  • SRCNN (2014)
  • • First DL model for SR
  • • 3 Conv layers: Feature Extraction → Non-Linear Mapping → Reconstruction
  • • End-to-end learning LR→HR
  • • Limitation: Shallow, limited receptive field
  • FSRCNN (2016)
  • • Faster version of SRCNN
  • • Operates on LR space directly
  • • Deconv layer for upsampling at the end
  • • 5x faster than SRCNN
  • VDSR (2016)
  • • Very Deep SR (20 conv layers)
  • • Global residual learning
  • • Learns residual: HR = LR + Residual
  • • Multi-scale: handles ×2, ×3, ×4
  • 6. GAN, Transformer & State Space Models
  • GAN-based: SRGAN & ESRGAN
  • • Introduced perceptual loss + adversarial training
  • • Generator produces HR, Discriminator evaluates realism
  • • ESRGAN: Residual-in-Residual Dense Blocks (RRDB)
  • • Sharper textures, high-frequency detail recovery
  • • Limitation: Training instability, mode collapse
  • Transformer: SwinIR
  • • Uses Vision Transformer with Shifted Window Attention
  • • Global feature learning via self-attention (Q, K, V)
  • • Excellent texture reconstruction quality
  • • Limitation: O(n²) complexity, high GPU memory
  • • Limitation: Struggles with very large input resolutions
  • State Space: SR-Mamba
  • • Based on Mamba / S6 selective state space models
  • • Linear O(n) complexity — very efficient
  • • Long-range dependency via state transitions
  • • Limitation: 1D sequential scan destroys 2D spatial context
  • • Limitation: Context loss at pixel boundaries
  • 7. Evaluation Metrics
  • MSE (Mean Square Error)
  • • Measures pixel-wise reconstruction error
  • • Formula: MSE = (1/mn) Σ (I(i,j) - K(i,j))²
  • • Lower MSE = better reconstruction
  • • Used in SRCNN, FSRCNN, VDSR
  • PSNR (Peak Signal-to-Noise Ratio)
  • • Measures reconstruction quality in dB
  • • Formula: PSNR = 10·log₁₀(MAX²/MSE)
  • • Higher PSNR = better image fidelity
  • • Inversely related to MSE
  • SSIM (Structural Similarity Index)
  • • Measures structural & perceptual similarity
  • • Compares: luminance, contrast, structure
  • • SSIM → 1: high similarity (good)
  • • SSIM → 0: poor similarity (bad)

  • 8. Why Hybrid? The Architecture Compromise
  • No single architecture solves all SR challenges
  • The Problem with Each Approach
  • The SRMamba-T Solution
  • CNN:
  • ✗ Limited receptive field (local only)
  • ✗ Cannot capture long-range dependencies
  • Transformer:
  • ✓ Global context via self-attention
  • ✗ O(n²) complexity — GPU memory explosion
  • Mamba (SSM):
  • ✓ O(n) linear complexity — very efficient
  • ✗ 1D sequential scan destroys 2D spatial info
  • Hybrid Insight: Sequence Before Space
  • Phase 1 — Mamba (Early Layers):
  • Dense feature extraction at linear cost O(n)
  • Handles the bulk of heavy computation efficiently
  • Phase 2 — Transformer (Later Layers):
  • Global refinement on already-compressed features
  • Un-windowed attention without memory explosion
  • → Best of both worlds: Efficient + Accurate
